## Contributing to Hiya Framework
Thank you for your interest in the Hiya Framework. This project, maintained by TaktikSpace.com, is built with a strict focus on efficiency, performance, and precision.

## Contribution Guidelines
- **Discuss Architecture Changes**: If you intend to make significant changes to the core architecture, please open an Issue first for discussion.
- **Code Quality**: We prioritize clean, efficient, and minimalist code.
- **License Compliance**: This project is licensed under the BSD 3-Clause License.  
  Contributors must retain the original copyright notice (TaktikSpace) and include the license text in all redistributions.  
  The names of the author or contributors may not be used to endorse or promote derived products without prior written permission.

## Pull Requests (PR)
- **Descriptive Branches**: Use a descriptive branch name for each feature or fix.
- **Clear Documentation**: Provide a clear description of your changes and why they are necessary.
- **Originality**: Ensure all submitted code is your original work.

## Standards & Ethics
We build this project with seriousness and integrity. We expect all discussions in Issues or Pull Requests to remain professional and technically focused.

If you have any questions, feel free to reach out via [taktikspace.com/hiya](https://www.taktikspace.com/hiya).
